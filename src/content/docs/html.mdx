### **什么是 html**


  
{/* type?: 'default' | 'warning' | 'danger' | 'info' */}
<Callout icon="💡" type="info">

html 不是一门编程语言,而是一种用于定义内容结构的标记语言;html 由一系列元素组成,这些元素用来包围不同的部分内容;一对标签可以实现内容的不同呈现;

</Callout>


### html 元素详解

<img src="/public/assets/images/web_introduction/grumpy-cat-small.png" alt="img" class="mt-4"/>


Opening tag: 开始标签;被大于号,小于号包围;表示元素从这里开始产生作用.
Closing tag : 结束标签;与开始标签类似,只是在元素名前多了一个斜杠.表示元素结尾
Content: 元素的内容;呈现的内容
Element : 开始标签、结束标签与内容相结合，便是一个完整的元素。

- 属性

元素也可以拥有属性，属性看起来像这样:

<img src="/public/assets/images/web_introduction/grumpy-cat-attribute-small.png" alt="img" class="mt-4"/>

**为元素提供额外信息,这些信息不会出现在实际内容中**

属性必须包含：

- 一个空格，它在属性和元素名称之间。如果一个元素具有多个属性，则每个属性之间必须由空格分隔。
- 属性名称，后面跟着一个等于号。
- 属性值，由一对引号（""）引起来。

**全局属性**

所有 HTML 元素共有的属性；它们可以用于所有元素，即使属性可能对某些元素不起作用。[全局属性列表](https://developer.mozilla.org/zh-CN/docs/Web/HTML/Global_attributes)

**特有属性**

有些属性是元素特有的,其他元素配置无效;例:meta 元素的 charset 属性,img 元素的 alt 属性等。

- 块级元素和内联元素

  块级元素:

  在 HTML 页面中以独立的块显示出来的元素，它们会占据一整行的空间，且默认情况下宽度会自动填满其父元素的宽度

  内联元素:

  在 HTML 页面中以行内的形式显示出来的元素，它们不会独立占据一行，而是与其他元素共同在一行内显示。内联元素的宽度由其内容决定，无法直接设置宽度和高度。

  **注:块级元素可以包含内联元素，但内联元素不能包含块级元素**

- 空元素

  不是所有元素都拥有开始标签、内容和结束标签。一些元素只有一个标签，通常用来在此元素所在位置插入/嵌入一些东西。这些元素被称为空元素;例:img 标签

```html
<img src="..." alt="图片信息" />
```

### html 基础结构

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body></body>
</html>
```

`<!DOCTYPE html>` 声明文档类型为 HTML5。浏览器将根据这一声明进行文档解析和渲染。

### 特殊字符

> 在 HTML 中，字符 `<`、`>`、`"`、`'` 和 `&` 是特殊字符

> 我们必须使用字符引用——表示字符的特殊编码，它们可以在那些情况下使用。每个字符引用以符号 & 开始，以分号（;）结束。

| 原义字符 | 等价字符引用 |
| -------- | ------------ |
| `<`        | `&lt;`       |
| >        | `&gt;`       |
| "        | `&quot;`     |
| '        | `&apos;`     |
| &        | `&amp;`      |

### 使用建议

**标题元素**

- 最好只对每个页面使用一次 `<h1>`——这是顶级标题，所有其他标题位于层次结构中的下方。
- 请确保在层次结构中以正确的顺序使用标题。不要使用 `<h3>` 来表示副标题，后面再跟 `<h2>` 来表示二级副标题——这是没有意义的，会导致奇怪的结果。
- 在现有的六个标题层次中，除非觉得有必要，否则应该争取每页使用不超过三个。

**超链接元素**

- 不要重复 URL 作为链接文本的一部分——URL 看起来很丑，当屏幕阅读器(视觉障碍人士)一个字母一个字母的读出来的时候听起来就更丑了。

  例: 👍https://example.com

  ❌https://example.com/j3k4lb2j9

- 不要在链接文本中说“链接”或“链接到”——它只是无用的内容。因为链接通常是用不同的颜色设计的，并且存在下划线

  例: 👍[下载 Firefox](https://www.mozilla.org/zh-CN/firefox/new/?redirect_source=firefox-com)

  ❌[点击这里](https://www.mozilla.org/firefox/)下载 Firefox

- 保持你的链接标签尽可能短——这非常重要，因为屏幕阅读器(视觉障碍人士)需要解释整个链接文本

- 尽量减少相同文本的多个副本链接到不同地方的情况。

**图片元素**

- 应该给你的图片取一个描述性的名称; 搜索引擎也读取图像的文件名并把它们计入 SEO。文件名：`dinosaur.jpg` 比 `img835.png` 要好。

- 不推荐使用绝对路径;这样做只会使浏览器做更多的工作，重新通过 DNS 再去寻找 IP 地址。

  ❌<img src="https://www.xxx.com/images/dinosaur.jpg" />

**嵌入元素**

- 使用 JavaScript 设置 iframe 的 `src` 属性;这使你的页面可以更快地被使用，并减少你的官方页面加载时间
- 始终使用`sandbox`属性;这样做的目的是为了保护主页面免受嵌入内容的潜在安全风险。如果不设置该属性，则`<iframe>`中的内容将具有与主文档相同的权限，即可以执行 JavaScript、提交表单或弹出窗口等操作。但如果设置了`sandbox`属性，则会禁用一些功能，例如 JavaScript 执行、表单提交、弹出窗口等，以及其他一些安全限制，比如禁止加载不安全的内容（http 协议）等。

如果绝对需要，你可以逐个添加权限（`sandbox=""`属性值内）;[更多属性值](https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/iframe#attr-sandbox)

**语义化**

- 使用`<strong>`元素而非`<b>`,使用`<em>`元素而非`<i>`

**其他**

不要在一个内联盒子中使用块级盒子;

```html
❌
<span>
  <div></div>
  <span></span
></span>
```

不要在 p 元素中使用任何块级盒子;

```html
❌
<p></p>
<div></div>
<p></p>
```

### 元素细节补充

1. 超链接元素

`文档片段`

超链接除了可以链接到文档外，也可以链接到 HTML 文档的特定部分（被称为**文档片段**）。要做到这一点，你必须首先给要链接到的元素分配一个 [`id`](https://developer.mozilla.org/zh-CN/docs/Web/HTML/Global_attributes#id) 属性。

```html
page1.html
<!-- 点击此链接将直接跳转至page2页面并且定位至#Mailing_address -->
<a href="page2.html#Mailing_address">跳转至公司邮寄地址</a>

page2.html
<h2 id="Mailing_address">邮寄地址</h2>
```

`download属性`

当你链接到要下载的资源而不是在浏览器中打开时，你可以使用 `download` 属性来提供一个默认的保存文件名。

```html
<!-- firefox-latest-64bit-installer.exe 下载文件名称 -->
<a
  href="https://download.mozilla.org/?product=firefox-latest-ssl&os=win64&lang=zh-CN"
  download="firefox-latest-64bit-installer.exe"
>
  下载最新的 Firefox 中文版 - Windows（64 位）
</a>
```

`电子邮箱链接`

当点击一个链接或按钮时，可能会开启新的邮件的发送而不是连接到一个资源或页面。这种场景可以使用 `<a>`元素和`mailto:` URL 协议实现。

```html
<!-- 发送电子邮件的窗口也会被用户的邮件客户端打开 -->
<a href="mailto:nowhere@mozilla.org">向 nowhere 发邮件</a>
```

2. 矢量图元素

   `两种不同的添加方式`

   **<img/>元素**

```html
<img
  src="equilateral.svg"
  alt="triangle with all three sides equal"
  height="87px"
  width="100px"
/>
```

优点:

- 使用熟悉的`<img>`元素语法，与其他图像类型相同。
- 可以通过将`<img>`元素嵌套在`<a>`元素中，轻松地将图像转换为超链接。

缺点:

- 无法使用 JavaScript 操作图像。因为 SVG 图像是基于 XML 的矢量图形，其每个元素都是一个 DOM 节点，而 JavaScript 在处理大量的 DOM 节点时可能会导致性能问题
- 如果要使用 CSS 控制 SVG 内容，必须为内联样式。而不能通过外部样式表进行引用。
- 不能用 CSS 伪类来重设图像样式（如`:focus`）。

  **<svg/>元素**

```html
<svg width="300" height="200">
  <rect width="100%" height="100%" fill="green" />
</svg>
```

优点:

- 减少 HTTP 请求，可以减少加载时间。SVG 内联到 HTML 文档中，可以避免浏览器额外发起请求来获取 SVG 文件，从而减少了 HTTP 请求，加快了页面加载速度。
- 可以使用 CSS 修改样式，实现更多的效果。SVG 元素可以与 CSS 混合使用，因此可以为 SVG 元素分配类和 ID，并使用 CSS 修改其样式，包括 SVG 特有的外观属性。
- 可以使用 CSS 交互和动画;在 SVG 图像上使用 CSS 交互和动画是非常方便的，通过 CSS 伪类如：hover、:focus 等
- SVG 元素可以包含在 a 元素中，从而成为一个超链接。

缺点:

- 将 SVG 内联到 HTML 文档中只适用于在一个地方使用的 SVG。如果需要在多个页面或位置复用 SVG，这种方法可能不是最佳选择。
- 将 SVG 内联到 HTML 文档中会增加 HTML 文件的大小，从而影响页面加载速度。因此，在使用内联 SVG 时应该尽量减小 SVG 代码的体积。
- 浏览器不能像缓存普通图片一样缓存内联 SVG。
- 对于老旧的浏览器需要做特殊处理

3. 图像元素

   响应式图片:根据分辨率大小显示相同的图片元素但大小会根据像素比例放大或缩小

```html
<!--
	备选图像源：elva-fairy-480w.jpg和elva-fairy-800w.jpg
	elva-fairy-480w.jpg被标记为480w，表示其宽度为480像素，而elva-fairy-800w.jpg被标记为800w，表示其宽度为800像素。
	
	sizes属性定义了两个条件：当视口宽度小于等于600像素时，图像的显示大小为480像素；当视口宽度大于600像素时，图像的显示大小为800像素。
	
	如果浏览器不支持srcset属性或无法选择合适的图像源，则会回退到src属性指定的默认图像源（elva-fairy-800w.jpg）
	alt属性用于提供图像的替代文本，以便在图像无法显示时进行替代显示。
	-->
<img
  srcset="elva-fairy-480w.jpg 480w, elva-fairy-800w.jpg 800w"
  sizes="(max-width: 600px) 480px,
	         800px"
  src="elva-fairy-800w.jpg"
  alt="Elva dressed as a fairy"
/>

<!--
	浏览器会根据设备的特性和条件自动选择最适合的图像源进行加载。浏览器会检查设备的像素密度、视口宽度等因素，并根据这些信息来决定使用哪个图像源。
	-->
<img
  srcset="elva-fairy-320w.jpg, elva-fairy-480w.jpg 1.5x, elva-fairy-640w.jpg 2x"
  src="elva-fairy-640w.jpg"
  alt="Elva dressed as a fairy"
/>

<!--
	<source>元素包含了两个备选图像源，并且每个<source>元素都有一个media属性，用于指定图像应该在哪些媒体查询条件下生效。
	<img>元素是<picture>元素中的默认图像源，如果浏览器不支持<picture>元素或无法选择合适的图像源，则会回退到<img>元素指定的默认图像源
	-->
<picture>
  <source media="(max-width: 799px)" srcset="elva-480w-close-portrait.jpg" />
  <source media="(min-width: 800px)" srcset="elva-800w.jpg" />
  <img src="elva-800w.jpg" alt="Chris standing up holding his daughter Elva" />
</picture>

<!--
	当需要根据设备特性和媒体查询条件加载不同图像时，使用<picture>元素。当需要提供不同分辨率的图像时，使用srcset属性。而sizes属性则用于指定图像在不同视口宽度下的显示大小。
	-->
```
